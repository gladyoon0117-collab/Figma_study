# 01_프레임과 그룹


## 1. 레이어
- 디자인 작업을 이루는 가장 기본적인 구성 요소로, 겹겹이 쌓이는 성질을 가짐.
- **계층 구조**: 좌측 레이어 패널 상단에 있을수록 화면상 맨 앞에 표시됨.
- **순서 조정**: 드래그 또는 단축키(`[` , `]`)를 활용해 앞뒤 관계를 설정.
- **피그마 레이어 종류**: 섹션, 프레임, 그룹, 컴포넌트, 텍스트, 이미지, 벡터, 오토 레이아웃, 마스크 등.


## 2. 프레임
- 배경 요소도 맞지만, 영역으로서 필요한 모든 곳에 사용되며 내부 콘텐츠를 관리하는 독립적인 객체.
- **핵심 역할**: 캔버스 내의 도화지 역할이자, 오토 레이아웃/컴포넌트/그리드 기능도 여기서 파생됨.
- **특징**:
  - 내부에 여러 콘텐츠를 포함할 수 있는 '상자' 개념.
  - **독립적 제어**: 하위 레이어가 없어도 프레임 자체로 존재하며 그 자체에 크기/스타일(Fill, Stroke 등) 지정 가능.


## 3. 그룹(Group) vs 프레임(Frame) 비교
두 기능 모두 자식 레이어를 묶어주지만, 작동 논리는 완전히 다름.

| 구분 | 그룹 (Group) | 프레임 (Frame) |
| :--- | :--- | :--- |
| **개념** | **끈**: 요소를 하나로 묶어만 둠. | **상자**: 독립적인 공간에 담음. |
| **아이콘** | 점선 사각형 (◌) | 우물 정자 (#) |
| **속성 상속** | 하위 레이어 속성의 합산 결과물. 내부 요소의 위치/크기 변경 시 그룹 영역이 즉시 상속되어 가변함. | 독립된 객체. 내부 요소와 무관하게 프레임 고유의 영역과 스타일(Fill/Stroke 등) 유지. |
| **스타일 제어** | 그룹의 Fill 변경 시 내부 모든 요소의 색상이 일괄 반영됨. | 프레임 스타일 적용 시 프레임 배경에만 반영되며 내부 콘텐츠는 보호됨. |
| **영역 제어** | 하위 레이어의 크기가 곧 그룹의 크기. | Clipping 기능으로 영역을 벗어난 자식 요소를 숨기거나 마스킹 처리 가능. |
| **리사이징** | 하위 요소가 **정비율**로 같이 변함. | 하위 레이어는 고정되고 **프레임 영역**만 변함. 하위 레이어에 맞춰 재설정하고 싶으면 우측 사이드바 `Layout` 패널 상단의 `Resize to fit` 아이콘 클릭. (`Option` + `Shift` + `Cmd` + `R`) |
| **생성** | `Cmd` + `G` (또는 우클릭 > `Group selection`) | `Cmd` + `Option` + `G` (또는 우클릭 > `Frame selection`) |

- **프레임<->그룹** : 우측 사이드바 패널에서 Group<->Frame으로 변경 가능.
- **해제(Ungroup)**: `Cmd` + `Shift` + `G` (프레임과 그룹 모두 동일하게 해제 가능)
- **아이콘 구분**: 레이어 패널에서 **# 모양**은 프레임, **점선 사각형**은 그룹으로 즉시 식별 가능.


### 상황별 도구 선택
- **그룹 추천**:
  - 수정이 많고 빠른 레이아웃 배치가 필요한 **기획/제안 초기 단계**에 유용.
  - **비율 보존**: 로고, 심볼 등 크기 조절 시 내부 요소의 정비율 유지가 최우선일 때.
  - **계층 구조 정리**: 기능적 제약(Constraints 등) 없이 빠르게 레이어 묶음이 필요할 때.
- **프레임 추천**:
  - 디자인 방향성과 기획이 확정된 **프로젝트 구축 단계**에서 화면을 구조화할 때 필수.
  - **구조적 독립성**: 상위 영역을 내부 요소와 무관하게 제어하거나, 프레임 자체에 스타일을 적용해 최적화할 때.
  - **반응형 설계**: 오토 레이아웃과 컨스트레인츠를 사용하거나 레이아웃 그리드를 적용해야 할 때.
  - **영역 제어(Masking)**: 영역을 벗어나는 요소를 마스킹(Clip Content)하거나, 의도적으로 외부에 노출시킬 때.
  - **인터랙션 구현**: 프로토타입에서 특정 영역의 스크롤 동작을 설정해야 할 때.
