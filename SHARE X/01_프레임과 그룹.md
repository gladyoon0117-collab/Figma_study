# 01_프레임과 그룹

## 1. 레이어
- 디자인 작업을 이루는 가장 기본적인 구성 요소로, 겹겹이 쌓이는 성질을 가짐.
- **계층 구조**: 좌측 레이어 패널 상단에 있을수록 화면상 맨 앞에 표시됨.
- **순서 조정**: 드래그 또는 단축키(`[` , `]`)를 활용해 앞뒤 관계를 설정.
- **피그마 레이어 종류**: 섹션, 프레임, 그룹, 컴포넌트, 텍스트, 이미지, 벡터, 오토 레이아웃, 마스크 등.

## 2. 프레임
- 배경 요소도 맞지만, 영역으로서 필요한 모든 곳에 사용되며 내부 콘텐츠를 관리하는 독립적인 객체.
- **핵심 역할**: 캔버스 내의 도화지 역할이자, 오토 레이아웃/컴포넌트/그리드 기능도 여기서 파생됨.
- **특징**:
  - 내부에 여러 콘텐츠를 포함할 수 있는 '상자' 개념.
  - **독립적 제어**: 하위 레이어가 없어도 프레임 자체로 존재하며 그 자체에 크기/스타일(Fill, Stroke 등) 지정 가능.
  - **클리핑(Clipping)**: 프레임 영역을 벗어나는 자식 요소를 숨길 수 있음 (`Clip content`).

## 3. 그룹(Group) vs 프레임(Frame) 비교
두 기능 모두 자식 레이어를 묶어주지만, 작동 논리는 완전히 다름.

| 구분 | 그룹 (Group) | 프레임 (Frame) |
| :--- | :--- | :--- |
| **개념** | **끈**: 요소를 하나로 묶어만 둠. | **상자**: 독립적인 공간에 담음. |
| **아이콘** | 점선 사각형 (◌) | 우물 정자 (#) |
| **속성 상속** | 하위 레이어 속성의 합산 결과물. | 하위 레이어와 무관하게 프레임 고유의 스타일 부여 가능. 독립된 객체. |
| **독립성** | 하위 레이어 삭제 시 그룹도 소멸. | 하위 레이어와 무관하게 고유 영역 유지. |
| **리사이징** | 하위 요소가 **정비율**로 같이 변함. | 하위 레이어는 고정되고 **프레임 영역**만 변함. |
| **생성** | `Cmd` + `Option` + `G` (또는 우클릭 > `Frame selection`) | `Cmd` + `G` (또는 우클릭 > `Group selection`) |


- **해제(Ungroup)**: `Cmd` + `Shift` + `G` (프레임과 그룹 모두 동일하게 해제 가능)
- **아이콘 구분**: 레이어 패널에서 **# 모양**은 프레임, **점선 사각형**은 그룹으로 즉시 식별 가능합니다.


### 상황별 도구 선택
- **그룹 추천**:
  - 로고, 심볼 등 크기 조절 시 정비율 유지가 중요할 때.
  - 레이어 계층만 빠르게 정리하고 싶을 때. (주로 기획/제안 초기 단계)
- **프레임 추천**:
  - **반응형 설계**: 오토 레이아웃, 컨스트레인츠를 사용해야 할 때.
  - **프로토타이핑**: 스크롤 영역을 지정하거나 화면 구조를 최적화할 때. (주로 구축/확정 단계)
 
-
  - 프레임 선택 시 활성화되는 우측 사이드바에 제공되는 프리셋 목록을 통해서도 역할 확인 가능.
 
    
 
    - 하위 레이어의 위치, 크기 변경 시 그룹은 변경한 하위 레이어의 속성을 그대로 상속 받아 그룹의 영역과 위치 변경됨. 프레임은 위치와 영역은 그대로.하위 레이어에 영향 받지 않음. 자체적으로 갖고 있는 clipping 기능을 통해 프레임을 벗어난 하위 요소를 클리핑할 수도 있음. (잘림) 이걸 그룹처럼 변경한 하위 레이어에 맞춰서 재설정하고 싶으면 우측 사이드바 (layout 패널의) resize to fit선택  or commang+shift+option+r.
    - 스타일 속성 변경 시
      - 그룹에 fill 보면 하위 레이어 값 보여짐. fill 변경해도 하위 레이어에 반영됨.

   
      - 프레임 : 크기 변경해보면 하위 레이어는 고정된 위치와 크기 유지. 프레임 크기만 변함. (근데 왜 변하는 경우가 있는 거지. 이럴 땐 cmd 누르고 프레임 크기 키우면 무시하고 프레임만 커짐. 근데 왜 변하는지????그룹처럼. 차이가 없음. 오토레이아웃도 안 걸려있고 constraints도 안 나타남)
    - 프레임을 그룹으로, 그룹을 프레임으로 변경하고 싶음 우측 사이드바 패널에서 group->frame으로 변경하면 됨.
  즉 그룹과 프레임은 중첩된 자식 레이어라는 공통점외에는 차이점 많음.
(독립적인 크기 조정, 스타일 적용, 영역 외 콘텐츠 포함, 콘스트레인츠로 리사이징, 오토레이아웃으로 리사이징, 레이아웃&그리드, 컴포넌트는 프레임만 갖고 있는 기능)
 
    - 프레임은 상위 영역을 독립적으로 제어하거나, 하위 요소의 크기를 통적으로 적용하고 싶을 때, 상위 영역을 벗어나는 하위 요소를 마스킹하고 싶을 때, 하위 요소를 상위 영역 외부에 위치시키고 싶을 때, 프레임에 스타일을 적용해 구조를 최적화하고 시플 때, 오토레이아웃을 사용해 부모 레이어의 크기를 동적으로ㅗ 적용하고 싶을 때, 프로토타입에서 스크롤 동작 원할 때, 레이아웃 그리드를 사용하려고 할 때 사용하면 유용 -> 디자인 방향성이나 기획이 확정된 프로젝트 구축 단계에서 프레임을 사용해 화면 구성.****
