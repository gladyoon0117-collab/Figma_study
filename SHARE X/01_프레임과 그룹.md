- 레이어
  - 디자인 작업의 기본적인 구성 요소. 겹겹이 쌓아 디자인을 이룰 수 있음. 텍스트, 이미지, 아이콘이 해당.
  - 피그마에서도 레이어 생성 시 좌측 레이어 패널에 순차적으로 쌓임.
  - 쌓이는 순서는 레이어 순서에서도 적용돼, 패널 상단에 위치한 레이어가 더 위에 표시됨. 드래그, 단축키 대괄호로 순서 조정 가능.
  - 피그마에서 레이어로 인식하는 요소 : 섹션, 프레임, 오토 레이아웃, 컴포넌트, 그룹, 슬라이스, 마스크, 타입, 이미지, 벡터
- 프레임
  - 주요 기능 중 하나. 오토 레이아웃과 컴포넌트도 여기서 파생.
  - 사용 시 레이아웃 그리드, 컨스트레인츠 등 추가 기능도 사용 가능해짐
  : 틀. 영역으로서 필요한 모든 곳을 이름. 내부에 여러 콘텐츠 포함할 수 있음.
  - 캔버스, 도화지의 역할
  - 프레임 선택 시 활성화되는 우측 사이드바에 제공되는 프리셋 목록을 통해서도 역할 확인 가능.
  - 기본 배경 요소도 맞지만 핵심은 영역으로서 필요한 모든 곳에 사용되고 내부에 여러 콘텐츠 포함할 수 있음.
  - 그룹과의 차이, 포함하는 방법에 있음.
    - 그룹 : 끈으로 콘텐츠 묶는 것.
    - 프레임 : 상자에 콘텐츠를 담는 것.
    - 우클릭 + frame selection or command+option+g.
    - 레이어 앞에 붙는 아이콘도 다름.
    - 그룹에 묶인 걸 밖으로 빼면 그룹도 사라지는데(하위 레이어 없이 단독으로는 존재 불가능), 프레임은 하위 레이어 분리해도 프레임 레이어 그대로 남아있음. 도형처럼 크기와 스타일 지정할 수도.
    - 레이어 자체가 객체이냐 아니냐, 스스로 형태가 있냐 없냐.
    - 그룹 우측 패널은 모두 하위 레이어 속성 합쳐서 나타낸 값.
    - 프레임 우측 패널은 프레임에만 해당하는 고유 값. 독립된 객체. 하위 레이어와 무관하게 설정 가능한 내용들.
    - 하위 레이어의 위치, 크기 변경 시 그룹은 변경한 하위 레이어에 맞춰 그룹의 영역과 위치 변경됨. 프레임은 위치와 영역은 그대로. 자체적으로 갖고 있는 clipping 기능을 통해 프레임을 벗어난 하위 요소를 클리핑할 수도 있음. (잘림) 이걸 그룹처럼 하위 레이어와 동일하게 재설정하고 싶으면 우측 사이드바 resize to fit선택  or commang+shift+option+r.
    - 스타일 속성 변경 시
      - 그룹에 fill 보면 하위 레이어 값 보여짐.하위 레이어에 반영됨.
      - 프레임은 스타일 넣으면 프레임 영역에 반영됨. 그 안에 콘텐츠가 아니라.
    - 크기 변경 시
      - 그룹 : 하위 레이어가 정비율로 같이 변함
      - 프레임 : 크기 변경해보면 하위 레이어는 고정된 위치와 크기 유지. 프레임 크기만 변함.
    - 프레임을 그룹으로, 그룹을 프레임으로 변경하고 싶음 우측 사이드바 패널에서 group->frame으로 변경하면 됨.
    - 그룹 : 레이어 계층 정리가 필요할 때, 크기 조정 시 정비율로 유지하는 요소(로고, 심볼 등), 상위 영역을 하위 영역에 따라 동적으로 적용하고 싶을 때 유용. -> 수정이 많은 프로젝트 제안 단계에서 많이 사용.
    - 프레임은 상위 영역을 독립적으로 제어하거나, 하위 요소의 크기를 통적으로 적용하고 싶을 때, 상위 영역을 벗어나는 하위 요소를 마스킹하고 싶을 때, 하위 요소를 상위 영역 외부에 위치시키고 싶을 때, 프레임에 스타일을 적용해 구조를 최적화하고 시플 때, 오토레이아웃을 사용해 부모 레이어의 크기를 동적으로ㅗ 적용하고 싶을 때, 프로토타입에서 스크롤 동작 원할 때, 레이아웃 그리드를 사용하려고 할 때 사용하면 유용 -> 디자인 방향성이나 기획이 확정된 프로젝트 구축 단계에서 프레임을 사용해 화면 구성.
