- 배리어블
  - 모든 종류의 디자인 속성 및 프로토타이핑 작업에 적용할 수 있는 재사용 가능한 값. 디자인 구축, 디자인 시스템 관리, 복잡한 프로토 등을 만들 때 드는 공수 줄일 수 있음.
  - : 변수. 피그마에서 변수=값. 재사용하고 싶은 속성을 변수로(bg, padding, label) 만들고 그에 해당하는 값 지정해 디자인 요소에 적용.
  - style vs variables
    - 모두 하나의 기준 역할. 디자인 전반에서 재사용 가능.
    - 스타일
      - 여러 값을 조합해 만드는 데 좋음.
      - 다른 스타일이나 변수엔 사용할 수 없는 내용.
    - 배리어블
      - 다른 스타일이나 다른 베리어블에도 적용 가능. 디자인 토큰?을 구현하는 데 도움됨.
      - 라이트모드, 다크모드 등 만들 수 있음.
      - 모드를 사용하면 베리어블에 재사용할 수 있는 범위도 넓어져 다양한 환경에 쓰인 디자인 구축 가능
    - 어떤 게 필요한 프로젝트 인지 검토, 사용

- 베리어블 종류
  - 4가지. color
    -  hex값 사용
    -  라이트, 다크 모드 같은 테마 작업하거나 브랜드 색상 팔레트 구성에 유용
    -  컬러 값을 입력할 수 있는 다양한 속성에 적용 가능
  - number
    - 숫자 값 사용
    - 정수, 소수점 이하 숫자도 사용 가능
    - 반응형 디자인과 다양한 언어간의 텍스트 속성을 처리하는 데 탁월
    - 객체 너비, 높이, 폰트 크기, 행간, 자간, 레이어 투명도 효과 등 숫자로 값을 입력할 수 있는 많은 영역에서 사용
  - string
    - hello같은 문자 값 사용.
    - 다구어를 지원하거나 다른 변수와 결합해ㅐ 텍스트 스타일 만들고 프로토에서 컴포넌트 변형을 전환하는 데 유용
    - 폰트 패밀리, 스타일, 텍스트 내용, 레이어 활성,비활성 등 문자열 입력하는...
  - boolean
    - 참과 거짓 값 사용해 특정 상황에 맞게 레이어 숨기고 표시하는 데 활용
    - 레이어 속성이나 컴포넌트 베리언트 프로퍼티가 t/f 값 갖고 있을 때 연결 가능

- 베리어블 등록
  - 좌측 사이드 바 variables> 창
  - 베리어블 창
    - 좌측 상단 콜렉션 영역 : 콜렉션 이름 수정, 삭세, 새로운 콜렉션 추가
    - 아래로는 생성한 베리어블의 그룹 목록 노출
    - create variable 클릭 > 베리어블 종류 노출 및 선택해 해당 영역에 베리어블 추가
    - 만든 베리어블 > 우클릭 > 세부 설정 편집 가능(베리어블 이름, 설명, 값 수정 가능. 어떤 속성에 사용할지 관리할 수 있는 scoping 영역있음. 코드와 연결할 수 있는 코드 구문 추가 가능. hide from publishing 활성화해 해당 베리어블을 라이브러리 퍼블리싱에서 제외 가능

  - 베리어블 그룹화
    - 묶음 베리어블 선택 > 우클리 > new group with selection 클릭 or 베리어블 이름 앞에 '그룹명 + / " 자동으로 그룹화
  - new variable mode: 모드 추가 가능 mode 1, 2

- 베리어블의 베리어블, 그룹, 모드 ,콜렉션 조합으로 원하는 디자인 시스템이나 환경 구축 가능
