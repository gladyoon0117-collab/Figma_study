# 04_오토 레이아웃 (Auto Layout)

## 1. 정의 및 핵심 가치
프레임에 추가할 수 있는 속성. 상위 프레임과 하위 콘텐츠들이 서로의 크기와 위치에 동적으로 반응하게 만드는 기능.

- **핵심 원리**: 상위 레이어(**컨테이너**)와 하위 레이어(**콘텐츠**)가 서로의 변화에 실시간으로 대응하는 구조 형성.
- **주요 효과**: 
  - 내용 및 구조 변경 시 레이아웃이 자동 재배치되어 동적인 디자인 구현 가능.
  - 디자인 시스템의 의도와 일관성을 유지하며 작업 공수를 획기적으로 절감.
  - 디자인 확정 후 **구축 단계**에서 실제 가변 상황을 대응하고 확인할 때 필수적으로 활용.
- **활용 예시**: 
  - 텍스트 입력값에 따라 너비가 자동으로 리사이징되는 버튼.
  - 하위 콘텐츠의 추가, 제거, 숨김 처리에도 기존 간격을 유지하는 컨테이너.
  - 상위 컨테이너 크기 변경 시 하위 콘텐츠가 자동으로 너비를 맞추는 반응형 구조.

### 생성 및 제거 방법
- **생성 (Add Auto Layout)**:
  - 적용할 레이어 선택 후 `Shift` + `A` 또는 우클릭 > `Add auto layout`.
  - 다수의 레이어, 자식 레이어를 가진 프레임/그룹 선택 후 우측 디자인 패널의 `Auto layout` 항목에서 `+` 버튼 클릭.
- **제거 (Remove Auto Layout)**:
  - 레이어 선택 후 `Option` + `Shift` + `A` 또는 우클릭 > `Remove auto layout`.
  - 우측 디자인 패널의 `Auto layout` 항목에서 `-` 버튼 클릭.

## 2. 리사이징(Resizing) 로직 및 제약 조건
상위 컨테이너와 하위 콘텐츠 간의 크기 상호작용을 정의. 실제 구현 환경과 동일한 구조를 만들어 디자인 화면에서 즉시 검증 가능.

| 구분 | 설정 값 | 의미 | 시각적 확인(Indicator) |
| :--- | :--- | :--- | :--- |
| **컨테이너** | **Fixed** | 내부 요소 변화와 상관없이 프레임 크기 고정. | - |
| **컨테이너** | **Hug** | 내부 콘텐츠 크기에 맞춰 프레임이 '안아줌'. | 패널의 W/H에 호버 시 무엇을 Hug하는지 표시됨. |
| **콘텐츠** | **Fixed** | 부모 컨테이너 변화와 상관없이 크기 고정. | - |
| **콘텐츠** | **Fill** | 부모의 남는 공간을 가득 채움(Container-Dependent). | 작업 영역 내 인디케이터로 범위 확인 가능. |

**논리적 충돌**: 콘텐츠 `Fill` + 컨테이너 `Hug` 조합은 서로를 기준으로 크기를 정하려 하므로 성립 불가한 조건. 이 경우 피그마는 자동으로 `Fill`을 `Fixed`로 변경.

### 크기 최소/최대 조건 부여 (Min/Max Width & Height)
무한정 늘어나거나 줄어드는 것을 방지하기 위해 수치적 제한을 두는 기능.

- **설정**: 컨테이너/콘텐츠 클릭 후 우측 패널 W/H 선택 → `Add min/max width/height` 클릭.
- **동작 예시**: `Min W: 80 / Max W: 360` 설정 시, 프레임을 늘려도 해당 범위 내에서만 작동.
- **주의 사항**: 컨테이너가 `Hug` 상태일 때, 내부 콘텐츠가 늘어나 `Max` 값을 초과하면 콘텐츠가 컨테이너 영역을 뚫고 나가는 현상 발생.
- **제거**: 우측 패널의 해당 설정에서 `Remove min and max` 클릭.

### 관계에 따른 간격(Gap) 동작의 차이
콘텐츠와 컨테이너의 리사이징 설정 조합에 따라 간격 조정 시 결과가 달라짐.

1. **콘텐츠 [Fill] + 컨테이너 [Fixed]**:
   - 간격(Gap)을 넓히면: 컨테이너 크기는 고정된 채, 내부 **콘텐츠의 크기가 줄어들며** 간격을 확보.
2. **콘텐츠 [Fixed] + 컨테이너 [Hug]**:
   - 간격(Gap)을 넓히면: 콘텐츠 크기는 유지된 채, 이를 감싸는 **컨테이너의 전체 크기가 커짐.**

## 3. 세부 레이아웃 및 고급 설정

### 방향, 간격, 정렬
- **Direction**: 콘텐츠 정렬 방향 지정 (`Vertical`, `Horizontal`, `Wrap`).
- **Gap & Padding**: 간격과 내측 여백 조절. `Individual padding`으로 상하좌우 개별 지정 가능.
- **Alignment**: 9개 포인트 기준 정렬. 콘텐츠 사이 간격을 `Auto`로 설정(또는 Alignment 패널 더블 클릭)하면 요소를 양 끝으로 밀어낼 수 있음.

### 고급 설정 (Advanced Settings)
- **Strokes**: Auto Layout 패널의 고급 설정에 들어가면 `Excluded`(테두리 제외) 또는 `Included`(테두리 포함 시 컨테이너 크기 가변) 설정 가능. 이 설정에 따라 컨테이너 크기 달라짐 (스트로크 포함해 컨테이너인 거라).
- **Canvas Stacking**: 콘텐츠가 쌓이는 구조 설정. `Last on top`(레이어 패널 최하단이 최상단 노출) 또는 `First on top`. 기본 설정은 `First on top`.
- **Text Baseline Alignment**: 활성화(`Active`) 시 서로 다른 크기의 텍스트들을 글자 밑줄(Baseline) 기준으로 정렬.

## 4. 특수 포지셔닝 기술

### 앱솔루트 포지션 (Absolute Position)
오토 레이아웃의 규칙을 무시하고 특정 위치에 요소를 고정할 때 사용. 콘텐츠에 절대적인 값을 부여한 포지션. (예: 알림 뱃지)
- **방법**: 콘텐츠 선택 후 `Absolute position` 클릭 → 위치 배치 → **Constraints**로 대응 방식 지정.

### 오토 레이아웃 무시하기
드래그 중 **`Space` 바**를 누르고 있으면 오토 레이아웃이 적용된 레이어 위에 다른 레이어가 내부로 빨려 들어가지 않고 겹쳐 놓을 수 있음. (예: 핸드오프 가이드 작업)

## 5. 실습: 터치 영역을 고려한 Row 설계
터치할 Row 영역 위아래의 구분선 사이 위 아래 여백을 터치했을 때도 클릭 동작 실행하게 만듦.

### 설계 로직 (Tag + Title + Date)

1. **태그 영역**: 태그 박스 크기는 가변적. > 그러나 레이아웃 유지(태그와 제목 사이의 간격은 일정)를 위해 프레임으로 한 번 더 감싸 `Tag Container`를 만들고 `Fixed Width`를 부여. > Row 전체를 `Hug`에서 `Fixed`로 변경한 뒤, 서비스의 전체 가로 너비(예: 360px)와 동일하게 맞춤.
2. **제목 영역**:  `Fixed`된 전체의 나머지를 제목이 공간을 Fill 하게끔 `Fill Container`로 남는 공간을 채우고, 텍스트 과다 노출 시 말줄임표가 작동하도록 한 뒤 `Max Width` 조건을 추가함 (Hug 설정은 계속 가져가야 하기 때문).
3. **터치 영역 확보**: 사용자의 클릭 미스를 방지하기 위해 Row 레이어에 충분한 상하좌우 **패딩(Padding)**을 적용하여 유효 클릭 범위를 확장함.
4. **최종 병합**: 구분선과 Row를 다시 오토 레이아웃으로 묶고, 구분선의 `Absoulute Position` 해제하고, 최종적으로 전체 테이블(Table) 단위의 오토 레이아웃을 적용하여 완성함.

- 오토 레이아웃 적용 시 캔버스 드래그보다는 레이어 패널에서 대상을 직접 선택해야 정확한 계층 구조를 잡을 수 있음.
