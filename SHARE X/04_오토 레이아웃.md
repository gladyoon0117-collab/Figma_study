프레임에 추가할 수 있는 속성.
사용 시 하위 콘텐츠와 상위 프레임이 서로에게 영향을 주어 동적으로 동작할 수 있게 만드는 기능
내용, 구조가 변해도 자동으로 레이아웃 변해서 이용해 동적인 디자인 만들 수 있음.
e.g. 텍스트 내용에 따라 리사이징되는 버튼, 하위콘텐츠의 추가,제거, 숨김 등의 변화에도 기존 구조 유지하는 컨테이너. 반대로 상위 컨테이너 크기 변경되면 하위 콘텐츠가 자동으로 대응하는 디자인
가변되는 콘텐츠 구조를 대응하고 확인할 때 유용. 작업에 들이는 공수 줄이고, 디자인 시스템과 의도를 일관되게 유지하게 하는 기능. 디자인 확정된 후 구축단계에서 유용하게 활용.

- 적용하고 싶은 레이어 선택 -> 우클릭 -> Add auto layout/shift+a or 다수 레이어나 자식 레이어 갖고 있는 프레임 또는 그룹을 선택해 우측 패널에서 +클릭해 추가 가능
- 제거하려면 우클릭 -> remove auto layout/shift option a or 우측 패널에서 -

- 상위레이어=컨테이너 하위 레이어=콘텐츠 서로 동적으로 대응하는 구조
- 디자인 패널
  - 컨테이너는 hug, fixed 설정 가능 콘텐츠는 fixed, fill 설정 가능
  - 컨테이너 fixed=하위 콘텐츠의 크기가 변해도 컨테이너 크기 고정으로 영향 받지 않음
  - hug= 하위 콘텐츠를 기준으로 크기 설정됨. 컨테이너 크기에도 영향
    - 디자인 패널에서 w/h hug에 호버하면 해당 프레임에 인디케이터 보여짐. 뭘 hug하고 있는지.
  - 콘텐츠 fixed: 컨테이너 크기가 변해도 영향 받지 않음.
  - fill : fill containeer. 콘텐츠가 컨테이너를 채움. 컨테이너에 따라 유동적으로 크기 변경됨. fill 역시 인디케이터 작업 영역에서 확인 가능.
 
- 크기 최소/최대 조건 부여
  - 컨테이너 클릭 후 우측 패널에서 w/h 선택 -> add min width/add max width / add min height / add max height -> e.g. min w 80 max w 360 -> 컨테이너 늘려보면 최대 360 최소 80까지만 늘어나거나 줄어듦.
  - 컨테이너 hug 상태에서 동일하게 하고 콘텐츠만 늘려보면 최댓값 넘어가면 컨테이너 뚫고 넘어감
  - 콘텐츠에도 설정 가능.
  - 설정한 값 지우고 싶으면 우측 패널 거기서 remove min and max
  - 실제 구현 환경과 동일한 구조 만들어서 디자인 화면에서도 확인 가능한 장점 있음.

- 세부 설정
- 방향 설정 : 콘텐츠 정렬 방향 지정
  - vertical layout : 세로 방향
  - horizontal layout :가로 방향
  - wrap : 가로 방향을 기준으로 컨테이너 넘어가는 콘텐츠를 자동으로 다음 열로 넘어가게 해주는 설정
- 간격 : 콘텐츠 간 간격(gap)
   콘텐츠와 컨테이너 간 간격(패딩)을 가로 세로로 지정 가능
  원한다면 individual padding 버튼을 눌러서 좌측, 상단, 우측, 하단 패딩 값을 개별로 지정 가능
  콘텐츠와 컨테이너 관계(fixed, fixed/fill, fixed...)가 어떻게 되어있느냐에 따라 간격을 변경했을 때도 다르게 동작함.
    - 콘텐츠 fill 컨테이너 fixed 일 때 간격 설정하면 콘텐츠가 간격에 따라 크기 바뀌고 컨테이너는 고정
    - 콘텐츠 fixed 컨테이너 hug는 컨테이너가 바뀜
    - !!!헷갈림:콘텐츠 fill 컨테이너 hug. 컨테이너 hug하면 콘텐츠는 fill-> fixed로 자동으로 바뀜. hug와 fill은 모두 고정값을 가진 컨테이너와 콘텐츠를 기준으로 동작할 수 있는 설정이라 두 설정 동시에 적용하는 것은 성립 불가한 조건.
 
- 콘텐츠 정렬
- 우측 auto layout 패널에서 alignment 하면 콘텐츠 정렬 변경됨. 콘텐츠 사이 간격을 auto로 설정해서 상하나 좌우, 끝과 끝에 위치시키고 싶으면 간격 auto or alignment를 더블클릭해서 설정 가능

- 스트로크
- 스트로크 쓴 부분을 포함해서 auto layout 영역을 적용할 건지 정하는.
- 패널 오레의 고급 설정으로 들어가면 strokes-excluded(스트로크 제외하는 설정)  included하면 컨테이너 크기가 달라짐.(스트로크 포함해서 컨테이너라고 생각하는 거니까)

- 캔버스 스태킹
- 콘텐츠가 쌓이는 구조를 정할 수 있는 설정. 레이어 계층 구조에서 마지막 레이어를 최상단에 올릴지 처음 레이어를 최상단에 올릴지 정할 수 있음. 기본 설정은 마지막에 있는 레이어가 최상단. e.g. 레이어 패널에서 위->아래 순으로 콘텐츠 1, 2, 3 레이어 있으면 콘텐츠 3이 디자인에서 가장 위에 놓임.
- 고급설정 -> last on top or first on top

- 앱솔루트 포지션
- 오레 적용된 컨테이너 내에 있지만 별도로 위치나 크기 대응이 필요할 때 사용가능한, 콘텐츠에 절대적인 값을 부여한 포지션
- UI에서는 뱃지 등을 표현할 때.
- 적용하고 싶은 콘텐츠 원하는 위치로 옮기면 오레 적용된 상태로 포함되는데 -> ignore auto layout 하고 원하는 위치로 옮기고 -> constraints로 위치 또는 크기 대응 방식을 지정해 놓으면 -> 컨테이너가 변하더라도 디자인의도를 유지하는 구조 만들 수 있음.
