# 05_컴포넌트 (Component)

## 1. 컴포넌트의 정의 및 구조
디자인 전반에 걸쳐 재사용할 수 있는 요소이자 기능입. 버튼, 아이콘 등 반복되는 UI를 **메인-인스턴스** 구조로 관리하여 수정 사항의 일관성을 유지하고 작업 효율을 극대화.

### 메인 컴포넌트 vs 인스턴스 컴포넌트
* **메인 컴포넌트 (Main Component)**: 모든 속성의 **원본**. 
    * 아이콘: 보라색 다이아몬드 4개(#)
    * 특징: 프로퍼티(Property)를 추가하여 다양한 옵션을 생성할 수 있는 권한을 가짐.
* **인스턴스 (Instance)**: 메인의 **복사본**. 
    * 아이콘: 빈 다이아몬드 모양(◇)
    * 특징: 원본과 연결되어 실시간으로 반영됨. 원본에 없는 요소를 추가/삭제하는 것은 불가능하지만 색상, 텍스트 등의 속성은 수정 가능함. (수정 시 해당 속성은 원본과의 연결이 끊어짐.)

### 생성 및 복제 방법
* **메인 생성**: 레이어 선택 → 상단 툴바 중앙 `Create Component` 클릭 (또는 우클릭 → `Create Component` 클릭 / `Cmd` + `Option` + `K`).
* **일괄 생성**: 여러 레이어 선택 → 툴바 드롭다운에서 `Create Multiple Components` 선택 → 한 번에 여러 개의 메인 컴포넌트 생성.
* **인스턴스 생성**: 
    * **Assets 패널**: 좌측 사이드바 Assets에서 드래그하여 복사하거나 Detail 창에서 `Insert Instance` 클릭.
    * **복제**: `Option` 드래그 또는 클릭한 채로 `Cmd` + `D` (길게 누를 시 대량 생성되므로 주의).
    * **복사/붙여넣기**: `Cmd` + `C` → `Cmd` + `V`.

## 2. 관리 및 수정 (Override)
인스턴스는 메인과 연결되어 있으면서도 내부 요소를 개별적으로 수정(Override)할 수 있음.

### 수정 범위 가이드
* **수정 가능**: 색상, 텍스트 내용, 레이아웃 그리드(`Shift` + `G`로 활성화), 중첩된 인스턴스 교체(Swap), 레이어 이름 변경.
* **수정 불가**: 레이어 순서(Z-index), 레이어 위치, 콘스트레인츠(Constraints) 설정.
    * **주의**: 핵심 구조 속성을 수정하면 원본과의 연결이 끊어지며, 이후 원본 수정 시 해당 속성이 반영되지 않음. 원복을 원할 경우 `Reset Instance`를 사용.

### 관리 패널 구성 (우측 상단)
| 구분 | 주요 기능 및 설정 내용 |
| :--- | :--- |
| **메인 패널** | **Component Configuration**: 이름 옆 설정 메뉴. 설명(Description) 및 기획 문서 링크 연결, 인스턴스 레이어 단순화(Simplify all instances) 설정. <br> **Property 설계**: 배리언트 및 각종 프로퍼티 추가/편집 영역 노출. |
| **인스턴스 패널** | **Navigation**: 원본으로 이동(`Go to Main Component`). <br> **Control**: 인스턴스 초기화(`Reset`), 메인에 덮어쓰기(`Push Changes`), 인스턴스 해제(`Detach`/ (`Cmd` + `Option` + `B`)). |

## 3. 프로퍼티 (Property): 효율화와 최적화
프로퍼티를 통해 불필요한 배리언트 생성을 줄이고 디자인 시스템을 최적화할 수 있음.

### 4대 프로퍼티 종류 및 설정 프로세스

#### 1) Variant (배리언트)
컴포넌트 내에서 형태, 컬러, 사이즈 등의 분류를 세트로 묶어주는 기능.
* **설정**: 메인 선택 → 우측 패널 `Properties` `+` → `Variant` 추가. (컴포넌트 하단 보라색 `+`로도 추가 가능)
* **결합 (Combine as variants)**: 이미 만든 여러 메인을 동시 선택 후 패널에서 `Combine as variants` 클릭하여 하나의 컴포넌트 세트로 통합. (이때 메인은 1개의 세트가 되고 그 안에 2개의 배리언트가 존재하게 됨.)
* **팁**: 일반 객체 상태에서도 여러 개 선택 후 `Create component set`을 누르면 즉시 생성됨.

#### 2) Boolean (불리언)
레이어의 노출(On-Off) 토글 제어. 활성/비활성 등 다른 타입의 컴포넌트를 쉽게 생성.
* **설정**: `Properties` `+` → `Boolean` 생성 → 이름 설정(예: show icon) → Value 설정 → `Create property`.
* **중요: 레이어 연결 (느낌표 해결)**: 생성 후 패널에 **느낌표(!)**가 뜨면 레이어에 연결되지 않았다는 뜻.
    1. 연결할 내부 레이어(아이콘 등) 클릭.
    2. 우측 패널 **Appearance** 옆 `Apply variable/property` 아이콘 클릭 → 생성한 프로퍼티 선택. (레이어 먼저 클릭 후 **Appearance** 옆의 아이콘 → `+` → `Create property`를 하면 생성과 연결이 동시에 되어 훨씬 직관적.)

#### 3) Instance Swap (인스턴스 스왑)
하위 레이어를 뒤지지 않고 패널에서 바로 다른 컴포넌트(아이콘 등)로 교체.
* **설정**: 메인 선택 → `Properties` `+` → `Instance Swap` 선택 → 이름/Value 설정 → 생성.
* **중요: 레이어 연결**: 교체할 하위 인스턴스 레이어 클릭 → 우측 패널 상단 `Create instance swap property` 클릭하여 연결.
* **Preferred Values**: 프로퍼티 편집창 하단에서 자주 사용하는 컴포넌트들을 체크해두면 수정 시 리스트 상단에 노출됨.

#### 4) Text (텍스트)
레이어 진입 없이 패널에서 텍스트 내용을 직접 수정.
* **설정**: 텍스트 레이어 개별 선택 → 패널 **Content** 옆 `Apply variable/property` 아이콘 클릭 → 프로퍼티 생성 및 연결.
* **용어**: **Name**(패널 노출 이름/라벨), **Value**(기본 입력 문구 내용).

## 4. 결과 및 실무 활용

### 인스턴스 프로퍼티 변경
* 원본을 복사한 인스턴스 클릭 시 우측 패널에 설정한 모든 프로퍼티가 노출됨.
* 왼쪽 레이어 탐색기를 뒤지지 않고도 **색상, 토글, Swap, 텍스트**를 즉시 수정할 수 있어 작업 속도가 비약적으로 향상.

### 프로퍼티 생성 시 주의사항 (느낌표 메시지)
* 프로퍼티를 패널에서 생성만 하고 실제 레이어에 **연결**하지 않으면 **느낌표(!)** 메시지가 뜸. 반드시 대상 레이어를 클릭해 **Appearance, Content, Swap** 아이콘 등을 통해 연결해주어야 옵션이 활성화됨.

## 5. 실습 및 해제 방법

### 컴포넌트 해제
* **컴포넌트 해제**: 메인은 직접 해제가 불가능하므로 `인스턴스 복제 → 원본 삭제 → 복제본 Detach` 순으로 진행하거나 전용 플러그인 `Detach Component` 활용.
* **인스턴스 해제**: 인스턴스 선택 → 우측 컴포넌트 옵션 → **`Detach instance`** 클릭 (단축키: **`Cmd` + `Option` + `B`**).

- 상단 앱바 (제목 유무 구분할 수 있는 컴포넌트 필요)
- 탭 (활성/비활성 상태 구분)
- 상품 구조 (베스트 랭킹 노출 여부를 제어할 수 있는)
