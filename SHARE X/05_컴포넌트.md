- 구성 요소. 디자인 전반에 걸쳐 재사용할 수 있는 요소이자 기능. 한 번 이상 반복적으로 사용하는 ui로 버튼, 아이콘 등 다양한 요소가 포함.
- 메인 컴포넌트(원본), 인스턴스 컴포넌트(복사본). 즉 메인 컴포넌트의 속성과 연결되어 메인 컴포넌트 속성이 변경될 경우 인스턴스에도 자동 반영.
  - 수많은 화면을 동일한 요소를 활용해 작업하는 ui 디자인에서 이 기능 사용하면 수정 사항 일괄 적용 가능. 반복 작업 줄여 효율 올릴 수 있음. 프젝 멤버들끼리 일관된 디자인할 수 있게

만들고 싶은 레이어 선택 -> 상단 툴바에서 중앙 create component 클릭해 메인 컴포넌트로 생성 가능.
or 레이어 선택 -> 우클릭 -> create componetn or command+option+k

    4. 원조 복붙 커맨드 c + v

메인컴포넌트와 인스턴트 컴포넌트는 다른 패널 가짐.
메인 컴포넌트
  컴포넌트 명 옆에 컴포넌트 설정 메뉴 (우측 상단 component configuration)
    : 설명과 관련 링크를 입력해 해당 컴포넌트 사용 방법에 대해 안내. 원할 경우 인스턴스 단순화하여 노출할 수 있는 기능
  프로퍼티 추가할 수 있는 영역 노출

인스턴스 컴포넌트
  다른 인스턴스로 교체할 수 있는 영역과 메인 컴포로 이동하는 버튼, 인스턴스 옵션 (다 우측 상단에 모여있음)
  인스턴스 옵션: 메인 컴포넌트와 연결을 해제하여 일반 레이어 상태로 되돌릴 수 있는 기능과 변경된 인스턴스 속성을 반대로 메인 컴포에 덮어 씌울 수 있는 기능. 변경된 인스턴스의 모든 속성 혹은 개별 속성을 리셋할 ㅜㅅ 있는 기능 제공
    - 인스턴스 수정 : 인스턴스 컴포는 내부 요소 수정 가능. 수정가능한 부분(디자인 관련된 내용들, 레이아웃 그리드(shift g로 활성화) 패널에서 margin 같은 값도 수정 가능)
    - 인스턴스 안에 중첩된 인스턴스(아이콘 등)도 교체 가능: 그 인스턴스 클릭한 다음 우측 패널 맨 위에서 드롭다운 펼침 -> 다른 아이콘으로 교체swap
    - 레이어명 변경 가능
  수정할 수 없는 건 : 레이어 순서나 Z 인덱스(??), 레이어 위치, constraints도 
  인스턴스 내에서 속성 수정하면 원본과의 연결 끊어져서 이후에 원본 수정돼도 반영 안 됨. (어떤 속성들???? 뭐든 바꾸면?? 글타고?)
    바꾼걸 메인과 똑같이 맞추고 싶으면 인스턴스 선택한 상태에서 우측 컴포넌트 옵션 펼쳐서 reset instance/specific changes

컴포넌트 해제
메인은 별도로 제공하지 않아서 인스턴스 컴포넌트로 복제 후 삭제하는 방법.
e.g. 메인 복제 -> 원래 메인은 삭제 -> 복사한 이 메인을 해제하면 메인 컴포넌트가 사라진 상태가 됨. detach instance
플러그인 사용 -> detach component -> 활성화하면 바로 해제됨.
인스턴스 해제하려면 -> 컴포 옵션 -> 디테치 인스턴스 단축키 커맨드 옵션 b

메인과 인스턴스의 차이
메인 : 원본. 모든 요소를 변경할 수 있으며 프로퍼티 추가하여 옵션 만들 수 있음
인스턴스 : 복사본. 원본에 없는 요소를 추가하거나 삭제하는 건 불가능하지만 색상, 텍스트 등의 속성을 수정할 수 있음. 인스턴스 내에서 속성 수정할 경우 원본과의 연결이 끊어짐.

프로퍼티
컴포넌트 효율적으로 활용할 수 있는. : 변경 가능한 속성. 컴포넌트 커스터마이징 가능. 다양한 상황과 요구에 유연하게 사용 가능해 작업 효율 향상
=집 짓기 (창문, 문,타일처럼 집 짓는 동안 여러 번 반복해서 사용하는 요소를 컴포넌ㅌ) (but 창문 색 각각 다를 수 있음. 창문이라는 메인에서 검정 창문 파랑 창문 큰 창문, 미닫이 창문 등의 인스턴스. 이러한 다양한 요구 만족시키기 위해 프로퍼티 사용.)
프로퍼티 통해 기본 창문에 여러 옵션 추가하여 각기 다른 조건과 상황에 맞게 창문 조정 가능
도표로 그리면 이런 식. 메인(기본 창문) - 세 갈래로 나뉨(컬러, 사이즈, 타입/각각 파란 창문, 큰 창문, 열리는 창문)
메인(버튼) - 초록 버튼, 큰 버튼, 비활성화 버튼
프로퍼티로 제어할 수 있는 항목 4개
- variant, boolean, instance swap, text
- 우측 사이드마 디자인 패널을 통해 제거 가능한 항목 파악하고 편집 가능
- 배리언트
  - 컴포넌트에서 파생된 다양한 컴포넌트를 세트로 묶어주는, 컴포넌트 내에서 분류 나눌 수 있는 기능
- boolean : 레이어 노출을 제어할 수 있는 속성. on-off 토글 사용. 활성 비활성. 다른 타입의 컴포넌트 쉽게 만들 수 있음
- instance swap : 디자인 패널에서 쉽게 다른 컴포넌트로 교체 가능. 하위 레이어 뒤지지 않아도 됨.
- text :
- 각 프로퍼티를 용도에 맞게 생성해 사용하면 불필요한 배리언트(? 생성을 줄여 관리 최적화 가능.

프로퍼티 추가/삭제
- 메인 선택 - 프로퍼티 플러스 - 선택
- 배리언트 프로퍼티
  - 프로퍼티 생성할 메인 컴포 선택 후 디자인 + 배리언트 추가 (이후에는 패널까지 안 가고 그 컴포 클릭하면 보라색으로 뜨는 플러스 선택을 통해 다른 배리언트 프로퍼티 추가할 수도 있음)
  - 우측 패널 add variant
  - 메인 컴포 여러 개 선택한 후에 -> 우측 패널에 combine as variants 버튼. 아하 내가 같은 버튼의 주황 초록을 미리 만들어두었으면 그걸 묶어서 메인 컴포넌트화? (그럼 메인컴포가 2개야?)
    컴포넌트 아닌 일반 객체일 때도 여러 개 동시 선택해서 배리언트 프로퍼티 적용된 상태의 컴포넌트 세트 만들 수 있음 (여러 개 선택 후 create component set)

- 불리언 프로퍼티
  - 레이어 활성 ,비활성을 제어하는 속성. 어떤 레이어를 제어할 건지와 함께 설정 필요. -> 메인 선택해서 프로퍼티 플러스 -> 불리언 추가 -> 이름 (show icon) - create property.  -> 추가 되었는데 느낌표. (컴포 안에 이 프로퍼티 사용하고 있지 않다는 메시지) -> 이 프로퍼티에 해당하는 레이어 연결해줘야 함!  -> 연결할 부분 클릭 -> 우측 패널의 appearance-> apply variable/property -> show icon 프로퍼티 선택
  - or 연결 레이어 클릭 후 appearance에서 apply variable/property  -> + -> create property
    (이 방법이 좀 더 직관적이라 실무에서는 이 방법 많이 사용)

- 인스턴스 스왕 프로퍼티
  - 컴포넌트 내 다른 컴포넌트 사용하고 있을 때 프로퍼티로 등록해서 쉽게 사용할 수 있는. 프로퍼티 플러스 -> instance swap -> but 마찬가지로 연결 필요.
  - 추가할 레이어 개별 클릭한 후 create instance swap property -> 크리에이트 프로퍼티
  - 추가한 스왑 프로퍼티 편집 버튼 -> 하단 preferred values + 자주 사용할 내용을 체크로 추가 가능 -> 해당 메인을 복사하고 스왑 편집 창에서 -> edit instance swap property보면 바로 확인 가능.

- 텍스트 프로퍼티
- property + -> text -> 연결할 텍스트 개별 선택 -> 컴포넌트 패널에서 content 옆에 apply vaiable/property 아이콘 -> 텍스트 선택
  근데 에디트 텍스트 프로퍼티 name value 각각 뭔지 모르겠음

- 인스턴스 컴포넌트 프로퍼티 변경
  - 컴포넌트 클릭 -> 패널 컴포넌트 영역에서 색상, 토글, swap,텍스트 다 바꿔볼 수 있음
  - 하위 요소 수정 시 레이어 탐색하지 않고도 패널에서 손쉽게 수정 가능.
