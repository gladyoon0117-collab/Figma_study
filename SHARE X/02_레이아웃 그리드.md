각 요소들의 정렬을 맞추는 게 중요. 그리드 규칙을 따르면 여러 명이 작업해도 일관된 디자인, 사용성 유지 가능
시각적으로 정리되어 보이게 하기 위해 줄이나 격자에 맞춰서 디자인하기 위한 도구로 격자라는 뜻의 grid 활용.
일반적으로 디자인 작업 전반에 걸쳐 동일한 그리드를 적용해서 그리드 시스템이라고 부르고, 피그마에서는 레이아웃 그리드 기능으로 이를 구현 가능
레이아웃 그리드
  - grid, columns, rows로 구성
  - 이를 조합해 아이콘, 페이지 레이아웃, 다양한 플랫폼에 대응 가능한 그리드 시스템 만들어 논리적이고 일관적인 디자인 유지
  - 해상도에 대응하는 그리드 시스템
    - 구성 : 오브젝트가 배치되는 컬럼, 여백 정의하는 거터, 마진 (화면의 좌우 맨 끝 여백 값인 마진이 디자인에 가장 큰 영향 -> 그리드 시스템 잡을 땐 좌우 마진 크기 영역 정하는 게 1순위 -> 이 마진 뺀 영역이 실질적으로 디자인 가능한 영역 -> 이 컨테이너 영역을 쪼개서 그리드를 만드는 게 추후 수정 줄일 수 있음)
      - 마진 정할 땐 여러 값을 가지고 콘텐츠 영역의 변화를 확인하며 다양한 디자인 테스트
      - 컬럼 : 글자나 이미지 같은 콘텐츠가 포함되는 영역
        - 컬럼 개수나 폭 설정은 디바이스 형태 or 브레이크 포인트에 따라 다르게 조절하며 사용
      - 거터 : 컬럼 사이의 간격. 컬럼이 가변이면 거터 수치 변경 시 단의 폭이 자동으로 조정됨

피그마에서 레이아웃 그리드
- 프레임에만 적용할 수 있음 (오브젝트, 그룹에서는 설정 불가. 프레임 레이어가 되면 우측 패널에 생김)
- +버튼 -> 추가 -> 적용된 그리드는 layout grid에서 눈 아이콘/shift+g로 노출 활성 비활성 제어 가능
