각 요소들의 정렬을 맞추는 게 중요. 그리드 규칙을 따르면 여러 명이 작업해도 일관된 디자인, 사용성 유지 가능
시각적으로 정리되어 보이게 하기 위해 줄이나 격자에 맞춰서 디자인하기 위한 도구로 격자라는 뜻의 grid 활용.
일반적으로 디자인 작업 전반에 걸쳐 동일한 그리드를 적용해서 그리드 시스템이라고 부르고, 피그마에서는 레이아웃 그리드 기능으로 이를 구현 가능
레이아웃 그리드
  - grid, columns, rows로 구성
  - 이를 조합해 아이콘, 페이지 레이아웃, 다양한 플랫폼에 대응 가능한 그리드 시스템 만들어 논리적이고 일관적인 디자인 유지
  - 해상도를 대응하는 그리드 시스템
    - 구성 : 오브젝트가 배치되는 컬럼, 여백 정의하는 거터, 마진 (화면의 좌우 맨 끝 여백 값인 마진이 디자인에 가장 큰 영향 -> 그리드 시스템 잡을 땐 좌우 마진 크기 영역 정하는 게 1순위 -> 이 마진 뺀 영역이 실질적으로 디자인 가능한 영역 -> 이 컨테이너(마진 뺀) 영역을 쪼개서 그리드를 만드는 게 추후 수정 줄일 수 있음)
      - 마진 정할 땐 여러 값을 가지고 콘텐츠 영역의 변화를 확인하며 다양한 디자인 테스트
      - 컬럼 : 글자나 이미지 같은 콘텐츠가 포함되는 영역
        - 컬럼 개수나 폭 설정은 디바이스 형태 or 브레이크 포인트에 따라 다르게 조절하며 사용
      - 거터 : 컬럼 사이의 간격. 컬럼이 가변이면 거터 수치 변경 시 단의 폭이 자동으로 조정됨

- 레이아웃 그리드 종류
  - 그리드에 레이아웃 가이드 추가. 10px 격자무늬 그리드가 기본 적용>세팅 아이콘>size=격자무늬 한칸의 크기를 설정하는 영역>10Px를 32로 바꾸면 개수도 크기와 동시에 달라짐
  - 컬러는 그리드의 색상값 설정. 색상, 투명도 변경 가능
  - 컬럼하려면>그리드 세팅>좌측 상단에서 컬럼 선택 가능>개수유형너비,마진,거터 설정 가능>타입: 컬럼 위치와 크기 등의 방식(stretch=설정한 컬럼 개수와 마진 거터값 바탕으로 너비가 변경되는 설정.즉 너비영역은 오토. 크기 변경하면 컬럼 너비도 같이 변경. 따라서 반응형 레이아웃에서 주로 사용하는 설정. / left right center 컬럼 위치를 좌우중앙으로 지정할 수 있는 영역)

- 레이아웃 그리드 중첩
  - 중첩 사용 가능해서 원하는 그리드 시스템 조합해서 만들 수 있음.
    -e.g. 반응형 모바일 프로젝트>일반적으로 모바일에서는 상단과 하단의 앱바가 고정으로 상시 노출. 컨테이너 영역에는 콘텐츠 고려한 레이아웃 필요. > 앱바:레이아웃 가이드>rows>1개>64px>top/ 생성된 그리드는 복제 가능해서, 복제할 그리드 선택한 뒤 command+D =복제/ 커맨드 씨,v도 됨. / 컨테이너 영역은 columns 개수 4개 마진 20px, 거터 16px> 앱바들과 구분되게 색깔 변경. > 3가지 레이아웃 그리드가 중첩돼 적용>동일하게 사용하면 스타일로 등록해서 더 편하게 사용가능.
- 레이아웃 그리드 스타일 등록, 적용>레이아웃 가이드의 스타일 아이콘>창에서 +해서 등록 가능>create style>로컬 파일에 종속되며, 해당 파일 내 모든 프레임에서 재사용 가능. 생성한 스타일은 edit로 수정 가능. 다른 프레임에도 library를 통해 등록된 스타일 빠르게 적용 가능.
