# UX 피그마 활용실무 개요

## 강의 소개
- UXUI 통합 프로세스 : UX Process (화면 설계) + UI Process (화면 디자인)
- UI와 별개로 UX 설계 업무를 주로 하거나 or 피그마 툴을 활용한 화면 설계를 작업하는 실무자 대상
- 에이전시에서의 UX 디자이너 업무 및 프로세스
  - 리서치 및 전략 도출 단계
    - 데스크 리서치
    - 아이데이션
    - 고객사 인터뷰
    - 전략 도출
    - 사용자 인터뷰
    - UX 디자인 리서치
    - 목업 제작
    - 컨셉 모델링
  - 구축 단계
    - UX 화면 설계서 작성
    - 프로젝트 일정 관리
    - QA 테스트
    - 커뮤니케이션
    - 오픈 준비

## UX 화면 설계 업무에 대한 이해
- 화면 설계서
  = Story Board(SB)
  - 실제 구현해야 하는 서비스에 대한 전체 화면과 함께 정책, 프로세스, 콘텐츠 구성, 기능들의 모든 정보를 담는 문서.
  - 필수 구성 요소
    - 이 문서가 어떤 내용을 담고 있는지 알 수 있는 표지
    - 버전별 어떤 수정 내용들이 있었는지 알 수 있는 히스토리 장표
    - 파트별 중간 인트로 장표
    - 공통으로 적용돼야 하거나 가이드가 필요한(예: 전체적 구조, 플로우) 이야기를 담은 공통 장의 장표
    - 화면 정의 장표 (왼쪽 와이어프레임 -> 넘버 매칭해 -> 영역별 정의를 오른쪽 description에 작성)
      - 와이어프레임 : 선(Wire)+틀(Frame). 선을 사용해 틀과 구조를 잡는 것.
    - 마무리 장표
  - 서비스 기획 및 설계하는 UX 디자이너 팀(전체 범위에서 메뉴 단위로 나눠 여러 명이 작업하는 방식)
    - 예: 01. 회원가입/로그인_SB_ver0.8_20240609 02. 홈 03. 카테고리 04. 검색 05. 상품 상세 06. 장바구니 07. 주문/결제 08. 이벤트/기획전 09. 좋아요/최근 본 10. 마이페이지 11. 설정
    - 00. COMMON GUIDE : 서비스 전반에서 공통으로 정의돼야 하는 것. (예: 기본 화면 구조, 스크롤 시 작동, 다양한 입력필드 케이스
    - 이후 UI 디자이너가 화면 디자인, 퍼블리셔와 개발자가 서비스 구현 및 개발. 밀착된 협업 중요.
      - 일관된 산출물을 전달하기 위한 충분한 내부적인 커뮤니케이션.
      - 하나의 Align으로 다시 맞추기 위한 불필요한 업무, 일정 차질, 비용 낭비 방지
      - 그래서 이런 정의 문서에는 모두의 고민이 같이 반영돼야 함.
