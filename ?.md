- 아이템 4*n의 구조
  - UI/UX 디자인이나 데이터 나열 방식을 말할 때 씀.
  - 한 줄에 아이템을 4개씩 배치하고, 데이터 개수에 따라 줄이 늘어나는 그리드(Grid) 형태
  - e.g. 화면 폭에 맞춰 한 줄에 4개씩 보여주세요.
- 모듈처리
  - 기능별로 부품화(모듈화) 하는 것
  - e.g. 이 기능은 나중에 다른 페이지에서도 재사용할 수 있게 독립적으로 만들어주세요.
  - e.g. ITunes API 호출 기능, 데이터 가공 기능, 화면 출력 기능을 각각 독립적인 부품처럼 나눠서 관리
- auto increment
  - db 용어
  - 자동 증가
  - 데이터가 새로 저장될 때마다 번호(ID)를 1, 2, 3... 순서대로 자동으로 하나씩 올려주는 기능
  - 데이터마다 고유한 정보가 자동으로 생성됨.
  - e.g. 햄버거집에서 주문 번호(250 → 251 → 252)는 시스템이 주문이 들어올 때마다 번호를 자동으로 1씩 더해서 생성했던 방식도. 주민등록번호 같은 역할. 순서대로 자동 증가.
